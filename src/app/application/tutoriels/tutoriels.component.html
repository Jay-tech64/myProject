Mini-tutoriel Angular

Qu'est-ce qu'Angular?

Angular est un framework front-end concu pour le développement d'applications dites "Single Page Application".

Voici les caractéristiques clés qui font d'Angular ce qu'il est :

  1. C'est un framework basé sur des composants

    2. Cela signifie que notre application finale va être composée de plusieurs petits composant lequels, ensemble, vont former notre application. Une analogie que j'aime bien faire pour expliquer ce concept est celle avec un casse-tête. Chaque morceaux du casse-tête (les composants), en eux-même, ne sont pas très significatifs. Par contre, lorsqu'on les mets tous ensembles, c'est là qu'on peut voir l'image globale souhaitée (notre application).

    Ce qui est génial avec les composants, c'est qu'on peut les réutiliser (ce qui veut dire moins de lignes de code). Un exemple de cette application de le fameux bouton « » de Facebook.

  2. Routage côté client

    3. Cette caractéristique va vous sauvé beaucoup de temps. En effet, avec Angular, il est possible de faire du routage côté client. Cela signifie que lorsqu'on clique sur un lien pour aller vers une autre page, on ne fait pas directement de requête au serveur. C'est Angular qui se charge d'aller chercher les informations qui nous faut pour la page Web désirée, et cela nous permet de sauver un temps fou!

  3. Injection de dépendance

    4. En gros, ce que l'injection de dépendance vous permet de faire, en tant que programmeur, c'est de vous faciliter énormément la vie lorsque vient le temps des tests unitaires. Par exemple, lors du test d'un composant, celui pourrait être dépendant d'un autre composant et/ou module pour bien fonctionner. C'est ici qu'Angular embarque. Il va faire en sorte que le composant testé ai tout ce dont il a besoin pour pouvoir faire les tests dont eu veut lui faire.

  4. Facilite la programmation orientée objet

    5. Grâce au langage de programmation TypeScript, les programmeurs Java se sentiront chez eux. En effet, les concepts d'héritage, de polymorphisme, d'encapsulation (private, protected, public, ...), d'abstraction (les interfaces) et de visibilité (package) sont rendus possible grâce à ce langage.

Mini-tutoriel Crud

Qu'est-ce que le Crud?

Le crud est un acronyme qui représente 4 opérations de requêtes HTML: CREATE , READ, UPDATE, DELETE.

C'est 4 opérations sont utilisés lorsqu'on fait appel à un WebService pour communiquer avec une base de données:

  1. CREATE

    2. L'opération CREATE permet de créer une nouvelle donnée et de l'insérer dans la base de données

  2. READ

    3. L'opération READ permet de lire une donnée dans une base de données

  3. UPDATE

    4. L'opration UPDATE permet de mettre à jour une donnée dans une base de données

  4. DELETE

    5. L'opération DELETE permet de supprimer une donnée dans une base de données

Les développeurs décident d'utiliser un WebService principalement pour protéger les données de la base de donnée. En effet, sans un WebService, la base de donnée serait visible à partir du navigateur et n'importe qui pourrait accéder à la base de données.

Mini-tutoriel Relation

Qu'est-ce que les relations?

Les relations sont les types de liaisons possibles entre les objets d'un diagramme UML

Voici les types de liaisons qui existent:

Formes Définition Code Typescript Remarques
AB Héritage class A extends B B est la classe parent
AB Implémentation class A implements B B est une interface
AB Association class A utilise B ou vice versa Il y a 3 types d'associations
AB Composition class A est un attribut de class B et doit être instancié pour tout constructeur de B Aucune remarque
AB Agrégation class A est un attribut de class B et l'instanciation est optionnelle pour tout constructeur de B Aucune remarque

Voici les types d'associations qui existent:

Formes Définition Code Typescript Remarques
AB OneToOne classe A est en relation 1 à 1 avec la classe B On a le choix entre A devient un attribut de B, ou l'inverse
AB OneToMany classe A est en relation 1 à plusieurs avec la classe B La classe A a un attribut de type tableau de B
AB ManyToMany classe A est en relation plusieurs à plusieurs avec la classe B On doit créer une nouvelle class AB qui aura au moins 1 attribut de chaque classes
